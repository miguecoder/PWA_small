Este es un ejemplo de los elementos que se deben tener en cuenta y agregar a un sitio web con responsive desing, para que pase a ser una PWA.

1. Una PWA permite ser descargada desde el navegador de un dispositivo, con la opción:

Para que esto ocurra es necesario que el sitio web cumpla con las siguientes condiciones:

  • - Debe tener activado un service worker.

  • - Debe estar desplegada por protocolo seguro https.

2. Las pruebas locales deben ser ejecutadas en un servidor (los service worker funcionan con el protocolo https y Localhost), cómo por ejemplo:

  • XAMPP: X (cualquier sistema operativo), Apache, MariaDB/MySQL, PHP, Perl.
  • MAMP: Mac OS, Apache, MariaDB/MySQL, PHP, Perl y Python.
  • LAMP: Linux, Apache, MariaDB/MySQL, PHP, Python.

4. Archivo manifiesto

- El paso a seguir es crear un archivo .json con el manifiesto de nuestra app web, a este archivo se le llamara manifest.json.

El manifest.json es un archivo JSON simple en su sitio web que le informa al navegador sobre su sitio web en el dispositivo móvil o computadora de escritorio del usuario. Chrome requiere tener un manifiesto para mostrar el mensaje Agregar a la pantalla de inicio.

El archivo manifest.json contiene detalles del nombre de la aplicación de su sitio web, los íconos que debe usar, el start_url en el que debe comenzar cuando se inicia y muchos otros detalles. Clic aquí para una explicación mas detallada.
